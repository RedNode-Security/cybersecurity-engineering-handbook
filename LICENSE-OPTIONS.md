# License Options

Choose a license before accepting external contributions.

Common options:

| License | Good for | Notes |
|---|---|---|
| CC BY 4.0 | Knowledge-base documentation | Allows reuse with attribution |
| CC BY-SA 4.0 | Documentation that should remain share-alike | Reusers must use the same license |
| MIT | Scripts and small tools | Very permissive software license |
| Apache-2.0 | Software with explicit patent grant | Common for larger engineering projects |
| All Rights Reserved | Private control | Not open source; external contributions are harder |

Recommended approach for this repository:

- Documentation: **CC BY 4.0** or **CC BY-SA 4.0**
- Scripts/code: **MIT** or **Apache-2.0**

A dual-license approach is reasonable:

- `LICENSE-DOCS.md` for Markdown documentation
- `LICENSE-CODE.md` for scripts and code examples

Do not merge outside contributions until the license model is clear.
